# Commit Guidelines (初期版)

使い始めるときのルール（あとで拡張可）

## 使う種類 (type)
- feat: 機能追加
- fix: バグ修正
- docs: ドキュメントのみ変更
- refactor: 挙動を変えない内部改善
- test: テスト追加・変更
- chore: その他（設定 / 依存 / CI など）

## 書き方（1行だけでOK）
type: 要約

例:
feat: add login page
fix: handle null user in session
docs: update README

(必要になったら本文や BREAKING CHANGE を追加予定)
